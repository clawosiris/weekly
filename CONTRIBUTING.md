# Contributing

Thank you for helping improve this project.

## Ground Rules

- Keep private OpenClaw runtime state, assistant memory, credentials, and logs out of this repository.
- Open an issue before large changes so maintainers can align on scope.
- Keep pull requests focused and reviewable.
- Include tests or verification notes for behavior changes.
- Follow the security policy for vulnerabilities instead of opening a public issue.

## Development

Project-specific setup instructions should be added once source code is imported.

## Pull Requests

Pull requests should include:

- A short summary of the change.
- Any linked issue.
- Risk notes for security, privacy, migration, or compatibility impact.
- Test results or a reason tests were not run.
